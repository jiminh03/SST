# SST
독거노인을 위한 홈 IoT 모니터링 서비스 웹 앱
개발 완료 이후 업데이트 예정 입니다

## 1. 개요

- **목적**: 독거노인의 생활 데이터를 실시간으로 모니터링하고, 이상 상황 발생 시 보호기관 직원이 원격으로 대응할 수 있는 웹 애플리케이션.
- **타겟 플랫폼**: **PC + 모바일 브라우저 (반응형 지원)**
- **개발 방향**: 하나의 코드베이스로 모바일·데스크톱을 동시에 지원.

---

| 구분 | 기술 |

| **테스트** | Jest, React Testing Library |

## 3. 주요 화면 및 기능

### 3.1. 로그인 화면

- **UI 요소**: ID, PW 입력 필드, 로그인 버튼
- **API**: `POST /api/auth/login` (JWT 기반 인증)
- **기능**: 토큰 로컬 스토리지 저장, 만료 시 자동 로그아웃

---

### 3.2. 어르신 목록 화면

- **목적**: 여러 어르신의 상태를 한눈에 확인
- **UI 요소**:
    - 카드형 리스트 (반응형: 모바일은 세로 스크롤, PC는 2~3열 그리드)
    - 카드 구성:
        - 프로필 사진, 이름, 나이, 호실
        - 최근 생활 데이터 요약
        - 상태 아이콘 (🟢 정상, 🟡 주의, 🔴 위험) → 경고등 스타일
    - 상단 필터 탭: 전체 / 정상 / 주의 / 위험
- **API**: `GET /api/elders`

---

### 3.3. 어르신 상세 화면

- **목적**: 선택한 어르신의 상세 데이터 및 대응
- **UI 요소**:
    - 상단: 프로필 + 상태 뱃지
    - 생활 데이터 카드 3종 (냉장고, 조명, 움직임) → 아이콘 + 수치 표시
    - 로봇 상태 (ON/OFF)
    - 버튼:
        - 방문 체크인/체크아웃 (파랑)
        - 응급 구조 요청 (빨강)
        - 보호자 연락 (파랑)
        - 메모 남기기 (회색)
- **API**:
    - `GET /api/elders/:id`
    - `POST /api/visit/checkin` / `POST /api/visit/checkout`
    - `POST /api/emergency/request`
    - `POST /api/elders/:id/memo`

---

### 3.4. 알림 화면

- **목적**: 이상 징후/위험 상황 알림 확인
- **UI 요소**:
    - 알림 리스트: 상태 아이콘, 메시지, 발생 시간
    - 클릭 시 상세 화면 이동
- **실시간 처리**: WebSocket (`/ws/alerts`) + 브라우저 푸시 알림
- **API**: `GET /api/alerts`

---

### 3.5. 스트리밍 화면 (조건부 활성화)

- **목적**: 로봇 카메라 확인
- **조건부 노출**: 평소엔 OFF → 터치 무응답 시 자동 ON
- **UI 요소**:
    - WebRTC 영상 뷰어
    - 제어 버튼(좌/우 이동, 음성 송출)
    - 액션 버튼: 응급 구조 요청, 보호자 연락
- **API**:
    - WebRTC 연결 `/api/stream/:elderId`
    - `POST /api/robot/control`

---

### 3.6. 기록(History) 화면

- **목적**: 어르신별 발생/조치 내역 확인
- **UI 요소**:
    - 타임라인 뷰
    - 필터(날짜, 상태)
- **API**: `GET /api/history?elderId=...`

## 4. 기능적 요구사항

### 4.1 사용자 인증

- 직원은 로그인 화면을 통해 계정 인증을 할 수 있어야 한다
    - 카카오는 좀 오바인 거 같음..
- JWT 기반 인증, 만료 시 자동 로그아웃

### 4.2 어르신 목록 관리

- 여러 명의 어르신을 리스트(카드형 UI)로 조회 가능
- 각 어르신 카드에는 이름, 나이, 주소?, 상태(🟢 정상 / 🟡 주의 / 🔴 위험) 표시
- 필터링 기능(전체, 정상, 주의, 위험) 제공

### 4.3 어르신 상세 조회

- 특정 어르신 클릭 시 상세 화면으로 진입.
- 상세 화면에는 프로필, 상태 뱃지, 생활 데이터(냉장고, 조명, 움직임), 리포트? 표시
- 로봇 상태(ON/OFF) 확인 가능
- 주요 버튼:
    - 방문 체크인/체크아웃
    - 응급 구조 요청
    - 보호자 연락
    - 메모 남기기

### 4.4 알림 관리

- 실시간 알림 수신(WebSocket) → 브라우저 푸시 알림 표시
- 알림 리스트에서는 발생 시간, 상태, 메시지를 확인 가능
- 알림 클릭 시 해당 어르신 상세 화면(카메라 스트리밍)으로 이동
- 카메라 스트리밍 ON → 어르신 상태 확인

### 4.5 스트리밍 관리

- 평상시 카메라는 OFF 상태.
- 터치 무응답 시 자동으로 스트리밍 ON.
- 직원은 스트리밍 화면에서 영상을 확인하고, 로봇 제어(좌/우 이동, 음성 송출) 가능.
- 긴급 상황 시 응급 구조 요청 버튼 실행.

### 4.6 기록(History)

- 모든 발생/대응 내역 자동 로그 기록
- 타임라인 뷰로 조회 가능
- 날짜별 / 상태별 필터링 기능 제공

---

## 5. 추가 UX 고려 사항

- 카메라는 평소 OFF → 이상 상황에서만 자동 ON
- 경고등 UI 색상 일관성:
    - 🟢 정상 = 초록 (#22c55e)
    - 🟡 주의 = 노랑 (#facc15)
    - 🔴 위험 = 빨강 (#ef4444)
- 모든 화면에 **응급 구조 요청 플로팅 버튼** 상시 노출
- 알림은 긴급도 순으로 정렬 (위험 → 주의 → 정상)
